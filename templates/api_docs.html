Tài liệu API

Version 1.0.0

Tài liệu đầy đủ cho các API của Hệ Thống Nhập Liệu Thông Minh

Base URL: http://localhost:55003

Xác thực

Hầu hết các API yêu cầu xác thực người dùng. Có hai phương thức xác thực:

Session-based

Sau khi đăng nhập thành công, server sẽ tạo session và trả về session cookie.

Các request tiếp theo cần gửi kèm cookie này trong header:

Cookie: session=<session_id>
OAuth 2.0

Hệ thống hỗ trợ đăng nhập thông qua Google OAuth.

Sau khi xác thực thành công, server sẽ tạo session tương tự như phương thức đăng nhập thông thường.

Khi xác thực thất bại, API sẽ trả về mã lỗi 401 Unauthorized hoặc 403 Forbidden.

Authentication APIs

POST /login

Đăng nhập vào hệ thống bằng email và mật khẩu.

Parameters
Tên Kiểu Bắt buộc Mô tả
email string Bắt buộc Email đã đăng ký của người dùng
password string Bắt buộc Mật khẩu của người dùng
remember boolean Không Ghi nhớ đăng nhập (mặc định: false)
Request Body 
{
  "email": "user@example.com",
  "password": "yourpassword",
  "remember": true
}
Responses
Status Description
200 OK

Đăng nhập thành công. Server sẽ trả về session cookie.

Set-Cookie: session=abc123; Path=/; HttpOnly
401 Unauthorized

Thông tin đăng nhập không chính xác.

{
  "error": "Email hoặc mật khẩu không chính xác"
}
400 Bad Request

Thiếu thông tin đăng nhập hoặc dữ liệu không hợp lệ.

{
  "error": "Vui lòng nhập email và mật khẩu"
}

Mã lỗi

Mã Tên Mô tả
200 OK Yêu cầu thành công
302 Found Chuyển hướng đến URL khác
400 Bad Request Yêu cầu không hợp lệ hoặc thiếu tham số bắt buộc
401 Unauthorized Chưa xác thực hoặc token không hợp lệ
403 Forbidden Không có quyền truy cập tài nguyên
404 Not Found Không tìm thấy tài nguyên
500 Internal Server Error Lỗi máy chủ nội bộ
